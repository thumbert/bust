Documentation: masked_demand_bids.rs

This page documents the HTTP endpoints exported from masked_demand_bids.rs and gives example requests and responses based on the unit tests in the file.

Endpoints

1) Get demand bids (range)

Path: /isone/masked/demand_bids/start/{start}/end/{end}

Method: GET

Description: Returns DA demand bids (and virtuals) between the date range [start,end]. Dates are civil dates (YYYY-MM-DD).

Query parameters

  • masked_asset_ids — optional, comma-separated asset ids (e.g. 28934,12345).
  • masked_participant_ids — optional, a single masked participant id (integer).
  • bid_types — optional, comma-separated bid types: INC, DEC, FIXED, PRICE.

Example (based on tests)

Reqwest (blocking) example used in tests (substitute RUST_SERVER):

GET {RUST_SERVER}/isone/masked/demand_bids/start/2025-06-01/end/2025-06-01?masked_asset_ids=28934&masked_participant_ids=504170

cURL example:

curl "http://localhost:8080/isone/masked/demand_bids/start/2025-06-01/end/2025-06-01?masked_asset_ids=28934&masked_participant_ids=504170"

Response shape

Array of DemandBid objects (fields):

  • masked_participant_id (u32)
  • masked_asset_id (u32)
  • bid_type ("INC"|"DEC"|"FIXED"|"PRICE")
  • hour_beginning (tz-aware timestamp string, e.g. 2025-06-01 00:00:00-04:00[America/New_York])
  • segment (u8)
  • quantity (f32)
  • price (f32)

Sample response (one item) — from test_get_offers:

[
  {
    "masked_participant_id": 504170,
    "masked_asset_id": 28934,
    "bid_type": "PRICE",
    "hour_beginning": "2025-06-01 00:00:00-04:00[America/New_York]",
    "segment": 0,
    "quantity": 19.6,
    "price": 992.0
  }
]

2) Daily aggregated zonal demand bids (MWh) — aggregated

Path: /isone/masked/demand_bids/daily/agg/start/{start}/end/{end}

Method: GET

Description: Returns daily aggregated zonal demand bids (MWh) for each participant over the date range. Aggregates over bid types FIXED and PRICE across load zones.

Query parameters

  • masked_participant_ids — optional, comma-separated participant ids (e.g. 504170,212494).

Example (based on tests)

GET /isone/masked/demand_bids/daily/agg/start/2022-01-01/end/2022-02-01?masked_participant_ids=504170,212494

Response shape

Array of rows:

  • day (DATE)
  • masked_participant_id (u32)
  • mw (f32) — daily MWh sum

Sample response (single item) — from test_get_daily_zonal_agg:

[
  {
    "day": "2022-01-01",
    "masked_participant_id": 212494,
    "mw": 8443.3
  }
]

3) Daily zonal demand bids (per asset)

Path: /isone/masked/demand_bids/daily/zonal/start/{start}/end/{end}

Method: GET

Description: Returns daily zonal demand bids aggregated per participant and per asset for the date range. Aggregates over bid types FIXED and PRICE.

Query parameters

  • masked_participant_ids — optional, comma-separated participant ids.

Example (based on tests)

GET /isone/masked/demand_bids/daily/zonal/start/2022-01-01/end/2022-02-01?masked_participant_ids=212494

Response shape

  • day (DATE)
  • masked_participant_id (u32)
  • masked_asset_id (u32)
  • mw (f32)

Sample response (single item) — from test_get_daily_zonal:

[
  {
    "day": "2022-01-01",
    "masked_participant_id": 212494,
    "masked_asset_id": 37894,
    "mw": 2549.5
  }
]

Notes & comments

  • The file also exposes internal helper functions used by the endpoints: get_demand_bids, get_daily_zonal_demand_bids_agg, and get_daily_zonal_demand_bids. The unit tests in the same file exercise those functions directly.
  • Some tests call internal functions directly (not via HTTP). Use the same query params when constructing requests to the endpoints.
  • Example payloads above are taken directly from the unit tests in the file.
